# Summary

When a retired refresh token is replayed, only that token is rejected; the rest
of the family remains valid, defeating replay protection.

# Steps to Reproduce

1. Log in and obtain refresh token R1.
2. Refresh to get R2 (R1 is now retired).
3. Replay R1.

# Expected Behavior

The entire token family is invalidated and R2 stops working.

# Actual Behavior

R1 is rejected but R2 still works.

# Technical Context

See authentication context: detecting reuse of a retired refresh token MUST
invalidate the whole family.
