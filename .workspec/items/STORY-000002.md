# Summary

Implement rotating refresh tokens with replay detection.

# Technical Context

See the authentication context: refresh tokens rotate on every use and a
detected reuse invalidates the entire family.

# Requirements

- Rotate refresh token on every `POST /auth/refresh`.
- Store only hashed refresh tokens.
- Invalidate the family on reuse of a retired token.

# Risks

- Blocked from completion by BUG-000001 (family not invalidated on reuse).
