# Coding Standards

- Prefer small, focused modules.
- Public functions are documented with a one-line purpose.
- No silent failures: errors are surfaced, logged, or returned — never swallowed.
- Tests accompany behavioural changes.

## Style

- 2-space indentation.
- `snake_case` in the database, `camelCase` in application code.
- Avoid premature abstraction; duplicate twice before extracting.
